# PREP法の説明と提案書への適用例

## PREP法とは

**P**oint（結論）→ **R**eason（理由）→ **E**xample（具体例）→ **P**oint（再結論）

「最初に結論を言い、理由と具体例で補強し、最後にもう一度結論で締める」論理構成。

## 提案書への適用

| 章 | PREP要素 | 内容の例 |
|---|---|---|
| 第1章 | P（結論） | 御社の課題と、私たちが提供できる解決策の全体像 |
| 第2章 | R（理由①） | 課題の現状分析・数値で示す深刻さ |
| 第3章 | R（理由②） | 既存の手段ではなぜ解決できないか |
| 第4章 | E（具体例） | 導入事例・実績・デモ |
| 第5章 | E（具体例） | 導入ステップ・スケジュール・費用 |
| 第6章 | P（再結論） | 期待できる成果・ROI・競合優位性 |
| 第7章 | P（行動喚起） | ネクストアクション・お問い合わせ |

## よくある構成の失敗

**失敗①：** 会社紹介を最初に長く入れる → 相手が飽きる前に課題・解決策を見せる
**失敗②：** 課題の深刻さを説明せずに解決策だけ提示 → 「なぜ必要か」の理由（R）が薄い
**失敗③：** 最後の章が「まとめ」だけで行動喚起がない → 「次に何をすれば良いか」を明示する
