# 比較軸の選び方とフレームワーク解説

## 基本の比較軸（用途別）

### 営業提案用（顧客への説明）
自社の優位性が際立つ軸を優先して選ぶ。

| 比較軸 | 何を比べるか |
|---|---|
| 価格・費用 | 初期費用・月額・総保有コスト（TCO） |
| 主要機能 | 顧客が最も重視する機能の有無 |
| サポート | 対応時間・専任担当の有無・SLA |
| 導入実績 | 業界別の導入社数・規模 |
| 使いやすさ | 習熟コスト・操作性 |

### 経営報告用（市場ポジション把握）
客観性を重視し、入手可能な事実情報で構成する。

| 比較軸 | 何を比べるか |
|---|---|
| 市場シェア | 業界内の占有率・成長率 |
| 価格帯 | ポジショニングマップ（安い↔高い） |
| ターゲット層 | 対象顧客の企業規模・業種 |
| 差別化ポイント | 各社が強調している独自優位性 |

## 比較軸を選ぶ3つの基準

1. **測定可能：** 「使いやすい」ではなく「操作習得に必要な時間（時間単位）」
2. **入手可能：** 公開情報・ヒアリング情報から裏付けられる項目のみ
3. **意思決定に直結：** 読む人が「で、どうする？」と思わない軸は外す

## 列数のガイドライン

- 競合2〜3社：4〜6軸
- 競合4〜5社：3〜4軸（多すぎると表が読みにくくなる）
